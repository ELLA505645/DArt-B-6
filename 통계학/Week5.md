### 11.1. 결측값 처리

*결측값이라는 것은 실제 분석 프로젝트에서 다루는 대부분의 데이터에 존재함.

*결측값 처리 전에 데이터 탐색을 통해 결측값의 비율이 어떻게 되는지, 한 변수에 결측값이 몰려 있지는 않은지 등을 파악한 후에 처리 방법을 결정해야함. 

~~~
결측값의 표시
NA / NAN 등
~~~

#### 완전 무작위 결측
결측값이 무작위로 발생한 경우
-> 결측값을 포함한 데이터를 제거해도 편항이 거의 발생하지 않음

#### 무작위 결측
다른 변수의 특성에 의해 해당 변수의 결측치가 체계적으로 발생한 경우

#### 비무작위 결측
결측값들이 해당 변수 자체의 특성을 가지고 있는 경우
ex. A마트의 고객정보 데이터, '고객 소득'변수에서 소득이 적어 값이 공개되지 않아 결측이 발생한 경우
*이때, 결측된 값은 실제로 값을 확인할 수 없으므로 비무작위 결측을 구분하기 어렵다. 


~~~
결측값 처리 방법
~~~
1. 표본 제거 방법
- 전체 데이터에서 결측값 비율이 10% 미만일 경우
     
2. 평균 대치법
- 결측값을 제외한 온전한 값들의 평균을 구한 다음, 해당 평균 값을 결측값들에 대치하는 것
- but, 관측된 데이터의 평균을 사용하기 때문에 통계량의 표준오차가 왜곡되어 축소되어 나타나고 p-value가 부정확해짐.
     
4. 보간법
- 데이터가 시계열적 특성을 가지고 있는 경우
- 단순 순서 보간법 vs 시점 고려 보간법
     <img width="550" height="198" alt="image" src="https://github.com/user-attachments/assets/787cc449-ee71-4e63-859e-3569b8090206" />
     
4. 회귀대치법
- 회귀식을 이용하여 결측값을 추정
- 추정하고자 하는 결측값을 가진 변수를 종속변수로, 나머지 변수를 독립변수로
- 결측된 변수의 분산을 과소 추정하는 문제가 있어, 인위적으로 확률 오차항을 추가하는 확률적 회기대치법을 사용하기도 함
  
5. 다중 대치법
- 단순대치를 여러 번 수행하여 n개의 가상적 데이터를 생성하여 이들의 평균으로 결측값을 대하는 방법
     1) 대치 단계
     2) 분석 단계
     3) 결합 단계
      <img width="409" height="202" alt="image" src="https://github.com/user-attachments/assets/2c333a6c-ac21-4477-8e81-e099d9783ca0" />
평균공식을 통해 각각 데이터셋의 상이한 추정치와 표준오차를 결합하여 결측치가 채워진 최종 데이터셋을 만들면 다중 대치법의 모든 단계가 완료됨.

### 11.4. 데이터 표준화와 정규화 스케일링

~~~
표준화, 정규화 스케일링
: 독립 변수들이 서로 단위가 다르거나 편차가 심할 때, 값의 스케일을 일정한 수준으로 변환시켜주는 것
~~~

#### 표준화
각 관측치의 값이 전체 평균을 기준으로 어느 정도 떨어져 있는지 나타낼 때 사용한다.
<img width="122" height="69" alt="image" src="https://github.com/user-attachments/assets/e8efe59c-6788-477a-98ec-9d99c25adde3" />
Zero-mean으로부터 얼마나 떨어져 있는지를 나타냄

#### 정규화
데이터의 범위를 0부터 1까지로 변환하여 데이터 분포를 조정하는 방법
*전체 데이 중에서 해당 값이 어떤 위치에 있는지 파악하는 데에 유용함.
<img width="181" height="54" alt="image" src="https://github.com/user-attachments/assets/9620d87d-e30b-4a25-8109-80adc8330628" />

<img width="621" height="224" alt="image" src="https://github.com/user-attachments/assets/3cbd92d7-87ee-4059-b75a-7c4aca9e3e91" />
-> 표준화와 정규화는 비슷한 듯하지만 다른 특성을 가지고 있기 때문에 필요와 목적에 따라 선택해야 함. 

*표준화, 정규화 방식은 이상치에 민감하다는 단점이 존재 -> 이를 보완하기 위해 RobustScaler도 많이 쓰임 ( 데이터의 중앙값을 0으로 잡고 Q1과 Q3 사분위수와의 1QR 차이를 1이 되도록 하는 기법.
